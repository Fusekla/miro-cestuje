# Product Bible

## One Sentence

Miro's Roadbook is a public travel-planning website that lets friends quickly understand an upcoming trip without digging through chats, notes, or messy AI drafts.

## Who It Is For

- primary audience: friends joining the trip
- secondary audience: Miro as planner and editor

The site is not meant for strangers, customers, or logged-in users. It is a polished trip-sharing page, not a travel product.

## Core Job

For any published trip, a visitor should understand:

- where the trip goes
- what kind of trip it is
- the pace of each day
- the rough flow of activities
- where the group sleeps that night

## Current Site Shape

### Home Page

The home page should:

- introduce the project in a warm, premium way
- list available trips as attractive cards
- make it obvious that each trip is a curated plan

### Trip Page

Each trip has one long page.

That page should include:

- a cover image
- a short trip introduction
- a compact facts block
- day-by-day entries

Each day should:

- show a short summary first
- open on click to reveal the detailed itinerary
- show accommodation name when available
- show drive time when relevant
- show a day highlight image

## Content Principles

- English only
- public-safe content only
- accommodation name only
- no hotel prices
- no booking links
- no weather
- plans only, not memory albums

## Experience Principles

- Make the trip easy to scan first and detailed only when needed.
- Keep writing practical, calm, and human.
- Prefer fewer stronger sections over many small widgets.
- Make the site feel intentional and elegant, not experimental or toy-like.

## Out Of Scope

Unless explicitly requested, the following are out of scope:

- databases
- live APIs
- admin editing interface
- user accounts
- comments
- itinerary collaboration tools
- expense tracking
- gallery or memory-journal features

## Privacy And Public Rules

- The website is public to anyone with the link.
- Content must be safe to expose publicly.
- Do not publish secret or sensitive information.
- Keep accommodation information limited to the name only.

## Success Criteria

The product is succeeding if a friend can open a trip page and answer these questions quickly:

1. What kind of trip is this?
2. What is happening on each day?
3. Which days are heavy, which are lighter?
4. Where are we staying?

If the page answers those well and still looks good on mobile, the product is doing its job.
